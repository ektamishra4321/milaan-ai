The system

Deterministic where money is counted.
ML only where judgment is needed.

Deterministic

Balance-integrity firewall

Opening balance + every transaction must equal closing — per row. A misparsed amount halts the run before it can poison a single match.

Deterministic

Four-tier matching engine

Exact → combined settlements → split payments → fee tolerance. Every match carries a rule-ID, score, and plain-language explanation. Zero ML in the money path.

Fine-tuned 3B

Local categorizer

Qwen2.5-3B + LoRA, trained on a free Colab GPU with a frontier LLM as teacher. Reads "UPI-SHRM TRDRS-…" into 15 categories at ₹0 per call.

LLM + guardrail

Exception explainer

Hypothesizes why a line didn't match — citing computable evidence — and must abstain when evidence is thin. It proposes; it can never alter a match.

Measured

An eval harness over everything

Matching graded against planted ground truth. Categorizer graded on a 1,000-question held-out exam — against its own teacher. Every rule and prompt change re-scored, so improvement is provable: this engine went from 1% precision (harness caught a real bug) to 1.000 in measured steps.
